DAO/DTO/VO

2017-11-28

클래스를 최소 4개로 나누어 코딩하는 객체지향(OOP) 수업을 진행하고 있다. 현재는 예) 회원 관리 : Main, Member, MemberDAO, MemberService 로 진행중인데 DAO 라는 용어를 접하면서 궁금즘이 생겨 찾아본 DAO, DTO, VO 개념. 위의 예 중에 Member가 DTO, VO에 해당한다. 인터페이스, DB 등을 배워야 완벽하게 이해할 수 있을 것 같다.

DAO

Date Access Object
데이터 접근을 목적으로 하는 객체(클래스) 이다. 커넥션 같은 거을 하나만 두고 여러 사용자가 DAO의 인터페이스를 사용하여 필요한 자료에 접근하도록 하는 것이 DAO의 개념이다. DB를 사용해 데이터를 조회하거나 조작하는 기능을 전담하도록 만든 객체(클래스)를 말한다.

DTO

Data Transfer Obeject.
데이터가 포함된 객체를 한 시스템서 다른 시스템으로 전달하는 작업을 처리하는 객체(클래스) 이다. DTO는 VO로 바꿔 말할 수 있는데 계층간 데이터 교환을 위한 자바 빈즈를 말한다.

VO

Value Object
DB 혹은 어떠한 물체 틀(형상화)을 잡고 이 틀을 잡기 위한 변수들을 모아 놓은 것이다. VO는 DTO와 동일한 개념이나 차이점은 read only의 속성을 가진다. (이하 DB 아직 안배웠으니 무슨 말인지 잘 모르겠지만 일단 메모) VO는 관계형 데이터 베이스의 레코드에 대응되는 자바 클래스이다. 형태는 DB 레코드를 구성하는 필드들을 VO의 Attribute로 하고 해당 변수에 접근 할 수있는 getter, setter 메소드의 조합으로 형성 되어 진 클래스 이다. 특성은 대체로 불변성이고 equals() 로 비교할 때 객체의 모든 값을 비교해야 한다.

Override

2017-11-30

헷갈리는 Overloading vs Overriding (오버로딩 vs 오버라이딩)

  • *Overloading(오버로딩) * : 메소드 오버로딩, 생성자 오버로딩이 있다. 같은 이름의 메소드를 여러개 정의하고, 매개 변수의 유형과 개수를 다르게 하여 다양한 유형의 호출에 응답하게 한다.
  • *Overriding(오버라이딩) * : 상위 클래스가 가지고 있는 멤버 변수가 하위 클래스로 상속 되는 것처럼 상위 클래스가 가지고 있는 메소드도 하위 클래스로 상속되어 하위 클래스에서 사용할 수 있다. 하지만 하위 클래스에서 메소드를 재정의 해서 사용할 수 있다. 상속 관계에 있는 클래스 간에 같은 이름의 메소드를 정의 하는 기술.
  • 오버로딩과 오버라이딩 성립 조건
구분 Overloading Overriding
메소드 이름 동일 동일
매개변수, 타입 다름 동일
리턴 타입 상관없음 동일
***
## @Override
- 상속 관계에서 메소드 재정의는 오버라이드(상속오라오라나에게오라…)
- 상속 관계인 경우 부모의 멤버(메소드)를 자식 클래스에서 재정의(Overriding)해서 사용할 수 있다.
- 재정의는 부모 메소드의 기능을 무시하고 자식 클래스에서 새롭게 구현하는 상태.
- 부모 메소드의 시그니처와 동일한 시그니처를 가진 자식 메소드를 구현하면 된다.
- 접근 제한을 더 강하게 오버라이딩 할 수 없다. (부모 메소드가 public 접근 제한을 가지고 있는 경우, 오버라이딩 하는 자식 메소드가 default나 private 접근 제한으로 수정할 수 없다.)
- @Override 어노테이션(annotation) 지정 가능
- 상속 지정이 없는 경우 자바의 모든 클래스는 java.lang.Object 클래스를 자동 상속 받는다.
- 이클립스 Source > Override > Implement Methods 메뉴 (부모 메소드의 시그니처를 잘 모를때 유용)
* 모든 클래스는 내부적으로 super, this 변수가 자동 생성되고 객체의 참조 주소가 자동 지정 된다.
* super : 상속 관계인 경우 부모 객체의 멤버(메소드) 접근시 사용하는 키워드로 자식 클래스의 메소드 내부에서만 사용 가능하다.
***
## final 키워드
* final 키워드는 불변의 특성을 가지는 클래스, 멤버(필드, 메소드) 선언시 사용. 결코 수정될 수 없음을 뜻한다.
* final 클래스는 상속 불가 클래스. 최종 클래스. 부모 클래스 지정 불가 상태.
* 대표적인 final 클래스는 자바 표준 API에서 제공하는 String 클래스 이다. 이 클래스 안에 있는 모든 메소드는 오버라이딩이 금지되어 있다.
* 부모 클래스를 상속해서 자식 클래스를 선언 할때 부모 클래스에 선언된 final 메소드는 자식 클래스에서 재정의 해서 사용할 수 없다. 오버라이딩 불가. 메소드 원본 기능을 유지한다.
***
## protected 키워드
* 접근제한자 public, protected, default, private 의 한 종류
* 같은 패지키 내에서는 접근 가능, 다른 패키지인 경우 상속관계일때만 접근 가능한다.
* 메소드 필드 등에 지정할 수 있다.

Scanner를 매개변수로

2017-11-27
  • 메소드의 매개변수에 Scanner 클래스를 넣어주면 Scanner를 통해 입력 받은 값을 매개 값으로 가질 수 있다. Scanner를 메소드의 매개변수로 지정한다는게 잘 이해가 안되었었는데, Scanner를 이용해 외부 입력 값을 받아서 메소드를 처리 한다고 생각하니 이해가 간다. 우리가 이용하는 Scanner는 java.util 패지키에서 제공하는 클래스이고 Scanner 객체를 생성해 사용한다. 한번 선언된 Scanner는 다른 클래스에서도 Scanner의 참조 주소를 공유하므로 한번만 선언하고 다른 변수명으로 사용할 수 있다.
  • 객체와 클래스의 관계에 대해서 이해하기 쉽게 설명해 놓은 문장을 찾았다.

    클래스(Class)는 객체(Object)를 만드는 방식입니다. 사실 대부분의 상황에서 클래스 = 객체라고 생각하셔도 무리는 없습니다.
    클래스는 객체를 만드는 방식중 하나이지만 JAVA에서는 객체를 만드는 방법은 클래스하나밖에(대부분 언어에서는 이거 하나) 없기 때문에 사실 둘은 어느정도 동의어라고 봐도 됩니다.
    하지만 차이점을 이야기하자면 위에도 설명했지만 클래스는 객체를 정의해 놓은것이죠.
    예를 들어서 컴퓨터가 필요할 때 컴퓨터 자체는 객체이고 컴퓨터를 만드는 방법(설계도같은 것)은 클래스라고 할 수 있습니다.
    클래스를 실제로 구현한것을 객체(Object)라고 부르며 이렇게 구현하는 과정을 인스턴스화(Instantiate)라고 합니다.
    그리고 이렇게 생성된 객체는 인스턴스(Instance)라고 부릅니다. *즉 Class는 Instantiate를 거쳐서 Object를 만드는데 그 Object는 그 Class의 Instance(구현물)이라고 부르는 것이죠.
    그냥 뭐 어렵게 생각할건 없구 중요하진 않을 수도 있지만 햇갈릴까봐 첨언하지만 보통 “야 이 인스턴스는~” 이라고 말하면 구현된 클래스, 즉 객체를 의미하고 “야 이 클래스는 ~”이라고 말하면 구현된걸 말한거일 수도있지만 개념적인부분까지 포괄한 것이라고 할 수 있죠. 그냥 간단하게 구현된 객체를 인스턴스라고 부른다고 생각하면되죠.

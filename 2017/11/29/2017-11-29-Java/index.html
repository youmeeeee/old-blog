상속

2017-11-29

상속이란?

  • 자식 클래스가 부모 클래스를 선택(상속)함으로써 부모의 기능을 물려받는 것.
  • 자바의 모든 클래스는 부모 클래스가 될 수 있다. 단 final 클래스는 제외.
  • 자식 클래스의 객체는 객체 생성시 부모의 멤버를 공유하게 된다. 부모 클래스의 객체가 자동으로 생성되기 때문이다.
  • 부모 클래스의 입장에서는 자식 클래스의 존재를 알 수 없다.
  • 자바의 모든 클래스는 java.lang.Object 클래스를 자동 상속 받는다.
  • Object 클래스는 자바의 상속 계층 구조에서 최상위 부모 클래스이다.
  • 상속을 잘 이용하면 새로운 클래스의 기능 구현시 기존 클래스가 구현하지 않은 부분만 구현하면 된다.
  • 개발 시간 절약, 클래스 재사용, 유지보수 시간 최소화, 기능 확장. 등등.
  • 상속시 부모 클래스의 public, protected 멤버만 상속된다. 단, 생성자는 제외. 일반적으로 public 메소드가 상속 대상이 된다. (?)
  • 같은 패키지의 경우 부모 클래스의 멤버중에서 default, public, protected 까지 멤버 상속 가능.
  • 다른 패키지의 경우 부모클래스의 멤버중에서 public, protected 멤버 상속만 가능.
    <형식> public class 자식클래스 extends 부모클래스 { }
  • 부모 클래스의 멤버를 호출하는 키워드는 super. 자식 클래스에서 오버라이딩(@Overriding) 된 값이 없을때는 this로도 부모 클래스의 멤버 접근 가능하기도 함.
  • 현재 객체(클래스)의 다른 instance 멤버 접근 키워드 this
  • 부모 생성자 호출
    • 부모 생성자는 자식 생성자의 맷 첫줄에서 호출 된다. 예를 들어 SubClass 의 생성자가 명시적으로 선언되지 않았다면 컴파일러는 다음과 같은 기본 생성자를 생성해 낸다.
    • 직접 자식 생성자를 선언하고 부모 생성자를 호출하고 싶다면 밑과 같이 작성하면 된다.
      super(parameters.. )는 매개값의 타입과 일치하는 부모 생성자를 호출한다. 만약 매개값의 타입과 일치하는 부모 생성자가 없을 경우 컴파일 오류가 발생한다. 부모 클래스에 기본 생성자가 없고 매개 변수가 있는 생성자만 있다면 반드시 자식 생성자에서 부모 생성자 호출을 위해 super(parameters…) 를 명시적으로 호출해야한다. 부모 클래스의 생성자 호출은 반드시 자식 생성자 가장 첫 줄에 위치 해야 한다.

컬렉션 프레임 워크(1)

배열의 단점(1. 크기 고정 2.객체를 삭제했을 때 해당 인덱스가 비게 되는 문제)을 해결하고 자료구조를 바탕으로 객체들을 효율적으로 추가, 삭제, 검색 할 수 있도록 java.util 패키지에 컬렉션과 관련된 인터페이스와 클래스들을 포함 시켜 놓았다.

Set 인터페이스를 구현한 클래스들 : HashSet, LinkedHashSet, TreeSet
List 인터페이스를 구현한 클래스들 : ArrayList, Vector, LinkedList
Map 인터페이스를 구현한 클래스들 : HashMap, HashTable, TreeMap, Properties

1. List

순서를 유지하고 저장, 중복 저장 가능하다.

ArrayList

  • ArrayList 객체 생성시 초기용량(capacity)는 10이지만 초과해도 자동증가 된다.

  • 모든 객체를 저장할 수 있는 장점은 있지만 저장할때 Object 타입으로 변환되어 저장되기 때문에 타입 파라미터 → generic 사용이 권장된다.

  • 객체 추가시 0번부터 차례대로 저장된다.

  • 중간 객체 제거시 바로 뒤 인덱스부터 마지막 인덱스까지 모두 앞으로 1 씩 당겨진다.

  • 반대로 특정 인덱스에 객체 삽입시 해당 인덱스부터 마지막까지 모두 1씩 밀려난다.

  • 따라서 빈번한 객체 삭제와 삽입이 일어나는 ArrayList를 사용하는 것이 바람직하지 않다.

  • ArrayList를 생성하고 런타임시 필요에 의한 객체들을 추가하는 것이 일반적이지만, 고정된 객체들로 구성된 리스트를 생성할 때는 하기 처럼 코딩 가능하다.

Vector

  • ArrayList와 동일한 내부구조
  • ArrayList와 다른 점은 동기화된(synchronized) 메소드로 구성 되어 있기 때문에 멀티스레드가 동시에 이 메소드를 실행할 수 없고, 하나의 쓰레드가 실행을 완료해야만 다른 쓰레드를 실행할 수 있다. → 멀티 환경 스레드 환경에서 안전하게 객체를 추가, 삭제할 수 있다. 스레드가 안전하다. (Thread safe)

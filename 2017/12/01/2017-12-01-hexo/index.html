Hexo 블로그 셋팅 완료

2017-12-01

드디어 꼬옥 마음에 드는 테마로 블로그를 바꿨다. 공부한 내용을 꾸준히 정리하기 위해서 시작하는 블로그이건만.. 셋팅은 완벽히 마음에 들게 하는데 시간이 너무 많이 걸리긴 했다…

  • Git이 너무 어렵다. Git 카테고리를 만들어서 조금씩 공부를 해나가야 할 것 같다.
  • jekyll과 다르게 Hexo의 경우 2개의 repository로 관리를 하는데.. 이걸 왜 이렇게 하는지 잘 모르겠다. 포스트 파일의 경우는 git에 push를 해주지 않아도 hexo generate/ deploy (hexo g -d) 만으로도 웹 블로그에 업로드 된다.
    • 동일한 저장소에 빌드 버전과 배포 버전을 브랜치로 나누어 관리 할 수도 있다.
    • hexo init 명령어를 이용해서 생성한 디렉토리를 잘 관리해야하는데, hexo deploy 명렁어를 사용하더라도 이 디렉토리 안에 있는 파일들이 모두 Github에 업로드 되는것은 아니다. /public 경로 안에 있는 파일들만 실제 Github에 업로드 된다. 2개의 컴퓨터에서 한개의 블로그를 운영하기 위해서 ‘hexo init’ 을 통해서 생성한 디렉토리를 일반적으로 개발 프로젝트 할 때 처럼 소스 저장소에 올려두고 관리를 해야하는데, 그것을 위해서 별도의 repository 가 필요하다.
  • 원하는 theme 폴더의 config 파일을 수정한 후 git 에 push하는 과정에서 번번히 테마 폴더의 이름을 바꿔주지 않고는 진행이 되지 않았다. add부터 안됨.
    • 테마를 적용할 때 Git 의 submodule을 활용하며 테마를 좀 더 깔끔하게 관리 할 수 있다.
    • submodule을 적용한 후 발행 환경(로컬)이 바뀌었을 때, 블로그 저장소만 복제하면 테마가 적용되지 않으므로 submodule로 설정한 테마를 따로 업데이트 해야한다.
      • hexo는내부적으로 테마를 기본적으로 지원하고 어떤 종류의 테마가 퍼블리싱 되어 있는지 확인도 할수 있다.
      • [Git의 submodule]https://git-scm.com/book/ko/v2/Git-%EB%8F%84%EA%B5%AC-%EC%84%9C%EB%B8%8C%EB%AA%A8%EB%93%88
      • [Git의 submodule를 사용하며 Hexo에 테마 추가하기]https://saystone.github.io/2017/04/04/How-to-add-a-theme-on-Hexo-using-submodule/
  • 결과적으로 이렇제 저렇게 만들었지만 어떤 구조로 돌아가는지는 전혀 이해하고 있지 못한 상황…

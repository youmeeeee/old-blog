프론트엔드 개발자 첫 직장에서 2년 (1편) - 나는 그동안 무슨 일을 했나

2020-08-18

입사 월이 8월인지라 막연히 이제 만으로 2년이 되었구나 하고 생각은 하고 있었지만 오늘이 딱 2년째 되는 날이였다. 길지 않은 시간 동안 팀이 급변 하는 과정을 몇 번이고 겪으면서 정신 없이 지냈지 싶다.

이 쯤에서 한번 내가 그동안 무엇을 해왔나 돌아 보고자 한다.

먼저 현재 소속되어 있는 팀에서 하는 일에 대해서 간단히 소개하기 위해 팀에서 개시한 구인 공고 글을 빌려 왔다.

저희 팀은 사업부가 수행하는 다양한 분야(금융 EMS, 데이터센터, 스마트팩토리, 스마트시티 등) 의 프로젝트에 사용할 제품을 개발하고 테크니컬 이슈를 해결하는 업무를 수행하고 있습니다. 팀명의 의미는 시각화 경험(Visualization eXperience)와 관련된 핵심 역량을 갖춘 팀이라는 뜻으로 주로 데이터 시각화와 사용자 경험이 중점인 제품을 다룹니다.

주요 제품은 데이터 시각화 프로젝트를 개발하는 툴 입니다. SQL, REST API 등을 이용해 데이터를 가져와 웹 브라우저에 웹 표준 기술로 표현 합니다. 2D, 3D 요소를 이용해 페이지를 디자인 할 수 있으면 자바스크립트를 이용해 UI를 구성 할 수 있습니다.

회사 업무다 보니 모든 것을 남길 순 없지만 간략하게 나마 팀 내에서 그동안 내가 했던 업무들을 정리해 보자. 시간 오름차순이다.

[1차 정리 - 2020. 08. 18]

템플릿 컴포넌트

추억의 수습 기간 첫 미션이다. HTML, CSS, JSON 문자열을 이용해 제품 내에 배치할 수 있는 2D 컴포넌트를 만드는 기능을 구현 했다. 제품의 코어 컴포넌트를 상속해서 구현 했다.

기술 스택

Vue, jquery, JqueryTemplate, pureMVC

라이선스 발급 페이지

부서에서 사용하는 구 + 현 제품들의 트라이얼, 정식 라이선스 발급을 위한 페이지를 개발 했다. 기존에는 Java 파일의 함수에 인자를 직접 넣어서 호출하는 방식으로 라이선스를 발급 하고 있었다. 해당 업무를 비개발자인 QA 파트에서 맡아서 진행하게 되면서 GUI 를 추가한 라이선스 발급 페이지를 개발 했다.

기술 스택

Vuejs

페이지 프로퍼티 변경

팀의 주요 제품인 시각화 프로젝트를 만드는 툴의 근간이 되는 것은 페이지 컴포넌트이다. 페이지 컴포넌트에 2D, 3D 컴포넌트를 배치하고 자바스크립트를 이용해서 화면에 데이터를 연동한다. 기존의 페이지 컴포넌트의 프로퍼티는 새로운 기능을 확장하기에 어려운 부분이 있었다. 페이지 컴포넌트의 기능 확장성을 위해서 페이지 프로퍼티를 구조를 대대적으로 변경하고 제품 전반에 걸쳐서 변경된 프로퍼티가 잘 동작하게 수정하는 작업을 진행했다.

기술 스택

Javascript, pureMVC

프로젝트 분리

하나의 제품은 여러 개의 프로젝트로 구성이 되어 있다. 유지 보수의 편의성을 위해 하나의 프로젝트에서 작은 단위의 프로젝트를 분리하는 업무를 맡았다. 해당 업무를 진행하며 axios interceptor, dev proxy, webpack에 대한 이해와 경험을 쌓을 수 있었다.

기술 스택

Javacript, webpack

사내 제품을 이용한 대시 보드 구현

외부에 공개된 사내 제품 캡처 이미지 참고

팀에서 개발 한 제품을 이용해 해외사업부에서 진행하는 프로젝트의 대시보드를 구현 했다. 화면과 데이터의 연동은 자바스크립트를 이용해 구현 한다.

매뉴얼 온라인화

기존에는 워드로 매뉴얼 제품 매뉴얼을 작성, 공유하고 있었다. 업데이트의 편의성과, 매뉴얼 이용에 대한 접근성을 높이기 위한 제품 매뉴얼의 온라인화 했다. 기존의 워드 파일을 마크다운 파일로 변환하고 변환 된 파일의 문자열을 마크다운 문법에 맞춰서 컨버팅 해주는 유틸을 만들어서 작업 했다. 해당 유틸에 jest를 이용해 TDD도 처음으로 적용해 보았다. 빌드와 배포는 vuepress와 firebase를 이용 했다.

기술 스택

javasscript, jest, vuepress, firebase

도면 편집 솔루션

팀에서 새롭게 개발하게 된 도면 편집 솔루션에서 일부 기능을 개발 했다.

  • 로그인
  • Vuex 스토어 중복 코드 개선
  • Axios를 활용한 API 요청 중복 코드 개선
기술 스택

vue, vuex

기타

이미 완성 된 제품을 기반으로 한 팀이나 보니 소소한 신규 기능 개발과 버그 수정 이슈는 지속적으로 하고 있다. (기본적인 SQL 작성을 통한 Java API 수정 포함)

내 경우 입사 전에 자바 위주의 교육을 받았지만, 취업은 프론트엔드 개발자로 하게 된 케이스이다. 객체 지향 프로그래밍에 대한 조금의 이해만 있으면 할 수 있을 거라는 전 팀장의 혜안이 있었다.

처음에는 자바스크립트에 대한 이해가 낮았기 때문에(거의 없었다..) 어려운 점도 없지 않았지만 지금은 프론트엔드 개발자로 커리어를 시작한 것에 매우 만족하고 있다.

다만 막상 지난 업무들를 정리 하고 나니 아쉬운 점들이 많이 보인다. 하나의 이슈를 임하는 마음가짐, 앞으로의 커리어에 대한 지속적인 고찰이 필요 할 것 같다.

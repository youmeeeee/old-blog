1. 데이터 타입

1-1. 기본형과 참조형의 종류 및 차이점

  • 자바스크립트 데이터 타입은 크게 두가지로 나뉜다.

  • Primitive Type (기본형) : 값을 그대로 할당한다.

    • Number
    • String
    • Boolean
    • null
    • undefined
    • symbol (ES6)

  • Reference Type (참조형) : 값이 저장된 주소 값을 할당(참조)한다.

    • Object
    • Array
    • Function
    • RegExp
    • Map, Set, WeakMap, WeakSet (ES6)

2-2. 함수선언문과 함수표현식

  • 함수선언문 (function declartion)

  • 기명 함수표현식 (named function exrpession) : 브라우저 엔진이 발전하면서.. 요즘엔 별로 쓸일이 없다.

  • 익명 함수 표현식(unnamed/anonymous function expression)

  • 함수선언문과 함수표현식의 차이는 할당에 있다. 할당을 하지 않으면 함수 전체가 호이스팅의 대상이 대고, 할당을 하게 된 경우는 함수는 그 자리에 남아있고 변수만 호이스팅을 하게 된다. (중요!!!)

  • ‘더글라스 크락포드’ 라는 아저씨는 함수선언문 대신에 무조건 함수표현식을 쓸 것을 권유하고 있다. 안전하고 예측 가능한 소스가 되기 때문이다. ES6에서는 이런 예측가능성이나 성능상의 이유로 function이라는 단어를 쓰지 않아도 되게 했다.

4. 클로저 (closure)

아직 뭔지 잘 모르겠음.

a closure is the combination of function and the lexical environment within which that function was declared.

클로저는 함수와 그 함수가 선언될 당시의 환경정보 사이의 조합이다. (최초 선언시의 정보를 유지!)

접근 권한 제어 / 지역변수 보호 / 데이터 보존 및 활용의 장점이 있다.

클로저로 private member 만들기

  • 외부로부터의 접근 제한
  • 전역 스코프의 변수를 최소화
클로저를 활용해서 private 멤버와 public 멤버를 구분하는 방법

  1. 함수에서 지역 변수 및 내부 함수 등을 생성한다.

  2. 외부에 노출 시키고자 하는 멤버들로 구성된 객체를 return 한다.

    → return 한 객체에 포함 되지 않은 멤버들은 private 하다.

    → return 한 객체에 포함된 멤버들은 public 하다.

5. prototype

prototype / constructor / __poroto__

생성자 함수가 있을 때, new 연산자를 써써 instance를 만들면 생성자 함수의 prototype이라는 프로퍼티가 instance의 __proto__라고 하는 프로퍼티에 전달이 됩니다.

즉, 생성자 함수의 prototype과 instance의 __proto__라고 하는 프로퍼티는 같은 객체를 참조한다.

그런데 __proto__ 는 내부 프로퍼티에 접근 할 때 이 단어를 생략할 수 있다.

즉 오른쪽 그림처럼 연결이 된것처럼 동작한다. 실제로 그런건 아니고 그런것 처럼 동작한다.

(부스트코스) HTML 내 JS 위치

2018-04-04

HTML 문서 안에는 HTML태그뿐 아니라 CSS, JavaScript코드가 여기저기 존재한다. JavaScript 코드는 body 태그 닫히기 전이나 body 태그가 끝난 직후 위치하는 것이 렌더링을 방해하지 않아도 좋고, css코드는 head 안에 위치해서 렌더링 처리 시에 브라우저가 더 빨리 참고할 수 있게 하는 것이 좋다.

  • JS가 head에 위치할 때

JS가 head에 위치하면 알람창이 먼저 뜨고 알람창을 끄고 나면 body내용이 출력된다.

  • JS를 body가 끝난 후 위치 시킬 때

엇 강의에선 크롬으로 테스트 했을 때 잘 됐는데.. 스크립트 코드를 body 밑에 위치시켜도 alert가 먼저 뜨고.. 그걸 꺼야 body내용이 보인다.

IE edge에서 테스트를 해보았다.

기대 했던 것처럼 body내용과 alert창이 동시에 보인다.

여러가지 이유로 예외가 있나보다….

위 설명처럼 js파일을 하단에 위치시키는 것이 일반 적이지만 defer / async 속성을 사용해서 선언한 곳과 스크립트 실행 시점을 분리 할 수 있다. (IE9까지는 지원되지 않는 속성이다.)

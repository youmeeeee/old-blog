(구멍가게코딩단) Part1-1. 개발 환경 설정

2018-04-10

PART 1 프로젝트의 기본 구조 구성

1. 개발 환경 설정

  • 스프링 웹 프로젝트를 시작하는 방법은 두가지가 있다.

    • Spring Boot를 이용하는 Srping Starter Project
      • 장점
        1. 별도의 설정이 필요없다.
        2. WAS 없이 실행이 가능하다.
        3. 로딩 시간이 짧아서 테스트 하기 편한다.
      • 단점
        1. 기존의 설정과 다른 방식으로 사용한다.
        2. JSP 설정 등은 별도로 해야한다.
    • Spring Template Project를 이용하는 Spring Legacy Project
      • 장점
        1. 현재까지 실무에서 많이 사용되고 있다.
        2. 다양한 자료가 존재한다.
        3. 기존 프로젝트를 이해하는데 도움이 된다.
        4. 모든 버전의 스프링에서 사용할 수 있다.
      • 단점
        1. 초반 데스트 환경 구성 등이 어렵다.
        2. WAS와 연동하는 경우 결과 확인에 많은 리소스를 소모한다.

  • 스프링 부트는 그 목적 자체가 최대한 간단하게 실행하고 배포가 가능한 수준의 웹 어플리케이션을 제작하는 것이다. 스프링 부트는 개발에 필요한 기본적인 모든 환경을 맞춰주고 최소한의 개발을 해야 하는 경우에 좋은 선택이 된다.

  • 프로젝트를 생성하면 Maven이라는 빌드 도구를 이용해서 필요한 jar 파일들과 프로젝트를 구성하게 된다. 프로젝트가 최초로 구성 되면 필요한 라이브러리들을 다운로드 받는데 시간이 필요하다. 가끔은 네트워크 상의 문제로 인해 구성시에 에러가 나는 경우가 있다. 그럴 경우 STS를 종료하고 (대부분의 경우) STS가 설치된 드라이브 밑에 사용자 이름 폴더 내부에 .m2 폴더(Maven이 다운로드를 받은 라이브러리를 사용하는 경로의 하위 폴더들을 삭제 후 재시작하는 방법이나 프로젝트 삭제 후 다시 생성하는 방법 등을 이용하여 다시 다운로드를 시도하는 것이 좋다.

  • Spring은 Maven을 내장하고 있다. Maven은 엄밀이 말해서 프로젝트 관리 도구로서, 프로젝트의 시작과 끝까지의 단계에 맞춰서 사용하는 개발 도구이지만, 가장 많이 사용되는 용도는 프로젝트에 필요한 의존적인 라이브러를 자동으로 관리해주는 용도로 많이 사용 된다.

  • Spring regacy project를 생성하고 나서 Run - Run on Server를 선택하면 WAS 설정이 이루어지 않은 상태에서도 기본적으로 제공 되는 서버를 이용해서 테스트를 진행한다. 별도의 코딩 작업 없이 스프링이 실행된느 것을 확인 할 수 있다.

  • 스프링 MVC 프로젝트의 템플릿 구조

STS가 위와 같이 별도의 설정 없이 웹 개발에 필요한 모든 경로를 만들어 주기 때문에 개발의 시간을 단축 시킬 수 있다.

  1. STS를 이용해서 Spring MVC 프로젝트를 생성하고 나면 JDK 버전은 1.6 버전을 기준으로 생성된다. JDK 1.7 이상의 버전을 사용하고 싶다면 해당 프로젝트의 Project Facets를 변경해줘야 한다.

  2. 스프링 프레임 워크의 버전 변경 (pom.xml의 수정 내용)

JDK랑 스프링 버전을 변경하고 나서 서버를 실행하면 페이지가 잘 안떠서 몇번이고 프로젝트를 다시 만들고 STS를 껐다가 .m2 폴더를 지웠다가 했다. (주의해서 천천히 하자)
3. Tomcat 서버 사용 설정 (기존에 이클립스에서 진행했던 것과 같아서 별도로 기록하진 않겠다.)

  • 스프링 프로젝트는 기본적으로 프로젝트 생성시 지정하는 패키지의 마지막 경로가 웹 어플리케이션의 중간 경로가 된다. 교재의 예제의 경우 ‘org.zerock.web’ 패키지를 작성했기 때문에 중간경로를 /web이 지정되므로, 최소 실행 경로는 브라우저 상에서 http://localhost:portNumber/web이 된다.

  • 설치 작업 정리

    • STS 를 이용해 Spring MVC 프로젝트를 다운로드 받아 생성한다.
    • JDK 버전 설정을 변경한다.
    • pom.xml을 이용해서 Srping 버전에 맞는 라이브러리를 추가한다.
    • Tomcat 서버를 설정에 추가한다.
    • 만들어진 프로젝트를 Tomcat 을 이용해 실행해 본다.

2018-04-02

MVC

MVC(Model, View, Controller) 패턴이란 개발 형태의 일종이다.

  • Model : DB와의 관계를 담당한다. 클라이언트 요청에서 필요한 자료를 DB로부터 추출하거나, 수정하거나 Controller로 전달한다.

  • View : 사용자한테 보여지는 UI화면이다. 주로 jsp파일로 작성하며 Controller에서 어떤 View Component를 보여줄 지 결정한다.

  • Controller : 클라이언트의 요청을 받고, 적절한 Model에 지시를 내리며, Model에서 전달된 데이터를 적절한 View에 전달한다.

MVC Model1

View와 Controlle가 같이 있는 형태이다.

  • 장점 : 단순한 페이지 흐름으로 개발기간 단축 /초보자에게도 쉬움 / 중소형 프로젝트에 적합
  • 단점 : 프로그램이 커질수록 유지보수가 어려움 / 디자이너와 개발자간 활발한 의사소통이 필요함
MVC Model2

최근은 대부분 모델2를 많이 사용한다. M/V/C가 모두 모듈화(부품화) 되어있는 형태이다. View는 어떠한 처리로직도 포함하지 않는다.

  • 장점 : 비지니스 로직과 뷰의 분리로 유지보수 확장이 용이 / 개발자와 디자이너의 작업이 분리 되어 분업이 편함 / 중대형 프로젝트에 적합
  • 단점 : 시간, 비용

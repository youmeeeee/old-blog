커밋 메세지 작성 규칙

2018-04-17

지금까지의 커밋 메세지는 엉망진창이다. 그냥 그 때 그 때 적당히 올렸었는데.. 오늘부터는 커밋 메세지 작성 원칙을 정하고 일관성 있게 커밋하자.

커킷 메세지는 제목 / 본문(선택) / 꼬리말(선택) 세 부분으로 작성한다.

제목

  • 타입
    • Feat : 새로운 기능을 추가할 경우
    • Fix : 버그를 고친 경우
    • Docs : 문서를 수정한 경우
    • Style : 코드 포맷 변경, 세미 콜론 누락, 코드 수정이 없는 경우
    • Refactor : 프로덕션 코드 리팩토링
    • Test : 테스트 추가, 테스트 리팩토링(프로덕션 코드 변경 없음)
    • Chore : 빌드 테스트 업데이트, 패키니 매니저 설정할 경우(프로덕션 코드 변경 없음)
  • 총 글자수는 50자 이내이며 마침표(.)를 붙이지 않는다.

본문(선택)

  • 커밋의 상세 내용
  • 제목과 본문 사이에 한줄 비우기
  • 한줄에 72자 이내
  • 한줄을 띄어 문단으로 나누거나 블렛을 사용하기

꼬리말(선택)

  • 이슈 트래커 ID

블로그의 마크다운 파일을 올리거나, 튜토리얼 보고 따라한 코드를 올리는 정도긴 하지만… 사소한 것부터 규칙대로 커밋하자!

스타일을 HTML 페이지에 적용하는 3가지 방법

우선순위대로

  1. inline
  2. internal
  3. external

일반적인 CSS위치는 css파일(external)에 두고 그 아래 style태그를 사용해서(internal 방식) 표현하기 때문에 그런경우 위에 나온 내용이 맞습니다.
하지만, internal과 external은 같은 우선순위를 가집니다.
같은 css내용이 있는 경우를 예로 보죠.

만약 css.css에서 div color를 blue로 주었다면, 뒤에 선언된 external방식의 css내용이 반영됩니다. 따라서 blue색깔로 나오겠죠.
즉 internal과 external은 같은 우선순위로 결정된다고 아셔야 합니다.

  • inline : HTML 태그 안에 넣기

    • 구조(HTML)와 스타일(CSS)이 섞이게 되므로 유지보수가 어렵다.

  • internal : HEAD에 <style></style>태그로 넣기

    • 구조(HTML)와 스타일(CSS)이 섞이게 되므로 유지보수가 어렵다.
    • 별도의 CSS파일을 만들지 않아도 된다.
    • 서버에서 따로 요청을 보내지않아도 된다.

  • external : 별도의 외부파일(.css)로 지정하기

    • CSS 코드가 아주 짧지 않다면 가급적 이 방법으로 구현하는 것이 가장 좋다.
    • <link rel="stylesheet" type="text/css" href="main.css" />

CSS상속 / 우선순위

상위에서 적용한 스타일은 하위에도 반영되, 여러 단계로 중첩된 엘리먼트마다 매번 같은 색상과 글자 크기를 부여하지 않아도 된다. 하지만 모든 CSS 속성이 이런 특징을 갖게 되면, 몇 가지 문제가 생긴다. 예를 들어 width 속성이 상속되면 하위 엘리먼트가 모든 같은 크기의 넓잇값을 가질 수 있다.이런 것은 원하는 것이 아니다… 그래서 box-model이라고 불리는 속성들(width, height, margin, padding, border)과 같이 크기와 배치 관련된 속성들은 하위엘리먼트로 상속이 되지 않는다!

우선순위

CSS는 여러가지 스타일 정보를 기반으로 최종적으로 경쟁에 의해서 적절한 스타일이 반영된다.

  • 동일한 css경우 나중의 것이 적용된다.
  • body > span{color:red;} 과 span{color:blue;} 중에 더 구체적으로 표현된것이 적용된다. red 승!
  • id > class > element 순으로 반영된다.
CSS Selector

HTML의 요소를 tag, id, html 태그 속성 등을 통해 쉽게 찾아주는 방법이다.

element에 style을 지정하기 위해서는 3가지 기본 선택자(tag / id / class)를 사용할 수 있다.

  • id는 특정한 1개만 쓰는게 좋다! #myId{color:red;}

  • class 셀렉터 사용 방법 .myClass{color:blue;}, div.myClass{color:green;}

  • 그룹 선택 가능
    h1, span, div{color:red;}

    h1, span, div#id{color:red;}

    h1.span, div.className{color:red;}

  • 하위요소 선택(공백)

  • 자식선택 (>)
  • n번째 자식 요소(nth-child)

Element가 배치되는 방법(CSS layout)

엘리먼트를 화면에 배치하는 것을 layout 작업이라고 하고 Rendering이라고도 한다.

엘리먼트는 기본적으로 위에서 아래로 순서대로 블럭을 이루며 배치된다.

다양하기 배치하기 위해서는 css의 중요한 추가 속성에 대해서 이해해야 한다.

  • display (block, inline, inline-block)
  • position (static, absoulte, relative, fixed)
  • float (left, right)

블록으로 싸이는 엘리먼트 (display:block)

display 속성이 block이거나 inline-block인 경우 그 엘리먼트는 벽돌을 쌓듯 블록을 가지고 쌓인다.

옆으로 흐르는 엘리먼트 (display:inline)

display 속성이 inline인 경우는 우측으로, 그리고 아래쪽으로 빈자리를 차지하며 흐른다. inline 속성 엘리먼트는 높이와 넓이를 지정해도 반영이 되지 않는다.

좀 다르게 배치시키기 (position 속성)

  1. 기본속성은 static 이다. 그냥 순서대로 배친된다.
  2. absolute는 기준점에 따라서 특별한 위치에 위치한다. (top / left / right / bottom 으로 설정한다.) 기준점을 상위 엘리먼트 단계적으로 찾아가는데 static이 아닌 positon이 기준점이다.
    **(absolute의 경우는 left / top 값을 0, 0 이라도 꼭 주는 것이 좋다.)
  3. relative는 원래 자신이 위치해야 할 곳을 기준으로 이동한다. top / left / right / bottom 으로 설정한다.
  4. fixed는 viewport(전체화면)좌측, 맨위를 기준으로한다.

absolute div는 기준점이 static이 아니라 positon으로 되어있는 wrap이 기준점이다.

간격을 다르게 해서 배치하기 (margin)

margin으로 배치가 달라질 수 있다. maring은 위/아래/좌/우 엘리먼트와 본인간의 간격이다. 따라서 그만큼 내위치가 달라진다.

기본 배치에서 벗어나 떠있기 (float)

하나의 블록 엘리먼트는 box 형태이다. (box-model)

블록 엘리먼트의 경우 box의 크기와 간격에 관한 속성으로 배치를 추가 결정한다. margin, padding, boder, outline으로 생선된다. (box-shadowe도 box-model에 포함지어 설명할 수 있다. box-shaodw는 border밖에 테두리를 그릴 수 있는 속성이다.)

엘리먼트의 크기는 부모의 크기가 기본이다.

block 엘리먼트의 크기는 기본적으로 부모의 크기만큼 가진다. 예를 들어 width:100%는 부모의 크기만큼을 다 가지는 것과 같다.

box-sizing과 padding

padding 속성을 늘리면 엘리먼트의 크기가 달라질 수 있다. box-sizing 속성으로 이를 컨트롤 할 수 있따. box-sizing 속성을 border-box로 설정하면 엘리먼트의 크기를 고정하면서 padding값만 늘릴 수 있다.

그래서 layout 구현방법은?!

  • 전체 레이아웃은 float를 잘 사용해서 2단, 3단 컬럼배치를 구현한다. 최근에는 css-grid나 flex 속성 layout을 위한 속성을 사용하기 시작했으니 브라우저 지원 범위를 확인해서 사용하면 된다.
  • 특별한 위치에 배치하기 위해서는 position absolute를 사용하고, 기준 점을 relative로 설정한다.
  • 네비게이션과 같은 엘리먼트는 block 엘리먼트를 inline-block으로 변경해서 가로로 배치하기도 한다.
  • 엘리먼트 안의 텍스트 간격과, 다른 엘리먼트 간의 간격은 padding과 margin의 속성을 잘 활용해서 위치 시킨다.

크롬 개발자 도구

  • 현재 엘리먼트의 값을 임시로 바꿀 수 있다.
  • 최종 결정된 css 값을 확인할 수 있다.

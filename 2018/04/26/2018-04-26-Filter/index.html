(최범균의JSP2.3) Filter

2018-04-26

필터란?

HTTP 요청과 응답을 변경 할 수 있는 재사용 가능한 클래스이다. 객체의 형태로 존재하며 클라이언트에서 오는 요청(request)와 최종 자원(JSP, 서블릿, 기타 자원) 사이에 위치하며 클라어은트의 요청 정보를 알맞게 변경 할 수 있다. 또한 최종 자원과 클라이언트로 가는 응답(response) 사이에 위치하여 최종 자원의 요쳥 결과를 알맞게 변경 할 수도 있다.

필터의 기본 구조

클라이언트와 자원 사이에 한개의 필터만 존재할 수 있는 것은 아니고, 여러개가 모여 하나의 필터 체인(chain)을 형성하게 된다.

필터의 구현

  • javax.servlet.Filter : 클라이언트와 최종 자원 사이에 위치하는 필터를 나타내는 객체가 구현하는 인터페이스
  • javax.servlet.ServletRequestWrapper : 필터가 요청을 변경한 결과를 저장하는 wrapper (클래스)
  • javax.servlet.ServletResponseWrapper : 필터가 응답을 변경하기 위해 사용하는 wrapper (클래스)

Filter Interface

필터 설정하기

필터를 설정하는 방법은 두가지가 있다.

(1) web.xml 파일에 관련 정보를 추가하는 방법

dispatcher태그는 실행되는 자원을 클라이언트가 요청한 것인지, 아니면 RequestDispatcher의 forward()를 통해서 이동한것인지, 아니면 include() 를 통해서 포함되는 것인지에 따라서 필터를 적용하도록 지정할 수 있다.

dispatcher 태그가 가질 수 있는 값

  • REQUEST : 클라이언트 요청인 경우 필터를 적용한다. (기본값)
  • FORWARD : forward()를 통해서 제어 흐름을 이동하는 경우에 필터를 적용한다.
  • INCLUDE : include()를 통해서 포함되는 경우에 필터를 적용한다.

하나의 filter-mapping에서 한 개 이상의 url-pattern 태그와 servlet-name 태그를 설정할 수 있다.

(2) @WebFilter 애노테이션을 사용하는 방법

Filter클래스가 @WebFilter 애노테이션을 가지면 자동으로 필터로 등록된다.

@WebFilter 애노테이션의 주요 속성

  • urlPattern : 필터를 적용할 때 URL 패턴 목록을 지정한다.
  • servletName : 필터를 적용할 서블릿 이름 목록을 지정한다.
  • fileName : 필터의 이름을 지정한다.
  • initParams : 초기화 파라미터 목록을 지정한다.
  • dispatcherTypes : 필터를 적용할 범위를 지정한다. 열거타입인 DispatcherType에 정의된 값을 사용한다. 기본 값은 DispatchType.REQUEST이다.

요청 및 응답 wrapper 클래스

필터가 필터로서의 기능을 하려면 클라이언트의 요청을 변경하고 클라이언트로 가는 응답을 변경할 수 있어야 한다. 요청과 응답을 변경할 때 사용하는 것이 ServletRequestWrapper와 ServletResponseWrapper이다.

  • 요청 정보를 변경하여 최종 자원인 서블릿/JSP/HTML/기타 자원에 전달한다.
  • 최종 자원으로부터 응답을 변경하여 새로운 응답정보를 클라이언트에 보낸다.

서블릿의 요청 래퍼와 응답래퍼를 만들려면 javax.servlet 패키지에 정의되어 있는 ServletRequestWrapper와 ServletResponseWrapper 클래스를 상속 받아서 구현해야한다. 하지만 대부분 필터는 HTTP 프로토콜에 대한 요청과 응답을 필터링 하기 때문에 이 두 클래스를 상속받아 알맞게 구현한 HttpServletRequestWrapper와 HttpServletResponseWrapper 클래스를 상속받아 구현하는 것이 좋다.

필터의 응용

사용자 인증 / 캐싱 필터 / 자원 접근에 대한 로깅 / 응답 데이터 변환(HTML 변환, 응답 헤더 변환, 데이터 암호화 등) / 공통 기능 실행 이 다섯가지 정도가 필터의 주요 응용 방식이다.

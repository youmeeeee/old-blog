FileIO

2018-04-03

서블릿을 배우다보니

Java FileIO 패키지 중에 PrintWriter 라는 객체가 존재한다. 어디다 뭘 써야할 때 가장 많은 메소드들을 가지고 있어 쓰기 편한 객체이다.

HttpServletResponse 객체의 getWriter() 메소드를 호출하면 PrintWriter객체를 리턴한다.

PrintWriter out = reponse.getWriter();

그리고는 이 객체의 print() 메소드를 사용해서 html을 출력 하거나 한다.

이거를 보다 보니까, 자바 시간 직렬화 할 때 잘 이해가 넘어갔던 FileIO 패지키에 대해 학습하고 싶어졌다.

File input and output

Stream이 뭔지도 이해가 잘 안됐었는데 IO에 대해 학습하려면 스트림에 대해 먼저 알아야 한다.

스트림은 1차원적인 데이터의 흐름을 의미한다. 자바는 이 스트림을 가지고 입출력을 처리하며, 흐름의 방향에 따라 입력 스트림, 출력 스트림, 데이터의 형태에 따라 문자 스트림(문자 혹은 문자열을 읽을 때 사용), 바이트 스트림(그림이나 실행 파일 등등)에 따라 나뉜다. 파입 입출력에서는 FileInputStream, FileOutputStrem, FileReader, FileWriter등 여러가지 스트림 클래스가 사용된다.

FileReader

텍스트 파일을 읽는 클래스이다. 출력의 대상이 텍스트 파일이고, 단위는 문자이다.

FileWriter

파일 쓰기

FileInputStream

파일로부터 바이트 단위로 데이터를 읽는다.

FileOutputStream

파일에 바이트 데이터를 쓴다.

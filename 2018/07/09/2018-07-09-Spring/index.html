web.xml 살펴보기

2018-07-09

익숙하지 않았던 스프링, 몇일 쭉 보다보니 조금씩 정리가 되긴한다.

오늘은 web.xml파일에 대해서 알아보자.

WEB-INF 밑에 web.xml은 deployment descriptor (배포서술자)의 구현체이다.

WEB-INF 폴더 아래 web.xml 파일명으로 항상 존재하며 서버 시작시 메모리에 로딩, 클라이언트 요청에 의해 컨테이너는 DD(배포서술자)를 참조하여 해당 서블릿에 맵핑한다.

현재 전자정부프레임워크를 학습하는 중이므로

전자정부 프레임워크에서 eGoveFramework Web Project를 만들었을때 생성되는 web.xml을 살펴보자. (eGovFrame 3.7)

위처럼 생겼다.

이전 스프링을 학습할 때 이 web.xml파일을 제대로 살펴본적이 없지만 찾아보니 크게 다르지 않을거라 생각한다.

하나씩 뜯어서 보자.

일단 web.xml의 루트 엘리먼트는 <web-app> 이다. 뭐 복잡해 보이지만 자동 등록된다. 해당 url 파일의 스키마 정의를 따른 다는 뜻이다.

다음은 <fliter> 엘리먼트를 보자.

필터는 웹 어플리케이션 전반 걸쳐 특정 URL이나 파일 요청시 먼저 로딩되어 사전에 처리할 작업을 수행(필터링) 하고 해당하는 유형 중 하나다. 주로 캐릭터셋 변경에 많이 사용된다.

<filter-mapping>에서 해당 필터를 적용할 URL패턴이나 servlet을 등록한다.

다음은 ContextLoaderListener로, 스프링 설정 정보를 읽는다. 이 때 1개로 충분하면 dispatcher에 저장하면 되지만 2개 이상일 경우 contextConfigLocation 초기화 파라미터에 설정파일 목록을 지정한다. listener는 반드시 등록해 놓아야 한다.

마지막으로 <servlet> (여기가 아직 제일 헷갈린다…)

DispatcherServlet 클래스를 초기화하여 spring의 servlet context를 생성한다.
servlet-mapping은 url pattern으로 지정된 값으로 웹 페이지 요청이 들어왔을 때 servlet-name에 지정되어 있는 이름의 servlet을 호출하겠다는 의미이다.스프링에서는 DispatcherServlet 이 모든 요청을 받고, 요청의 URL과 맵핑하는 Controller에 위임하는데 예를 들어 Controller class에 @RequestMapping(“/test”) 으로 annotation(어노테이션) 지정되 있는 메서드가 있다면 URL://URL주소/test.do 요청시 DispatcherServlet 에 해당하는 URL과 mapping 되는 것을 찾은 후에 해당 메서드에 요청한다.

이 외에에 아래의 내용 등들도 DD에서 설정이 가능하다. (그 밖에도 더 있음)

  • Session이 유지되는 시간 설정 (분 단위)

  • 에러 페이지 처리

  • welcome file list (웹 애플리케이션 요청 시에 시작파일을 지정)

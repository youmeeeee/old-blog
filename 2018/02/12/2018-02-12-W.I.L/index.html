느낀 점

  1. JDBC프로젝트는 DB담당, 코딩 담당으로 나눠서 하고 있는데 나는 코딩 담당이다. 옆에 앉은 친구가 DB 담당이라 둘이 도와가면서 하고 있다. DB 담당은 아니지만 쿼리 날리는게 좀 익숙해진 기분이다. 각종 함수 쿼리, 서브쿼리는 아직 좀 어렵지만.. JOIN하고 VIEW만드는거 위주로 연습이 많이 된 것같다. 제약사항에 관한 이해도도 높아졌다. 다른 조 프로젝트도 좀 도와주면서 하느라 내가 해야할 코딩은 예상보다 밀리고 있는 상태다…

  2. 크롱의 HTML, CSS 세미나를 다보고 느낀건 역시 HTML, CSS는 따로 공부하지 말자! 이다. 중학교때 연예인 팬페이지 만들고 운영하면서 부터 접했던 HTML.. 그렇게 생각하니 나와의 역사가 20년 가까이 대는구나.. 그때는 아이프레임으로 공지 같은걸 메인 화면에 띄우고 제로보드 게시판이랑 연결하고 그랬었는데.. 제로보드 설치 따라하고 그랬던 기억도 새록새록.. 그게 나와 웹의 첫 인연이였네.(갑자기 혼자 시간여행ㅋㅋ) 그 때도 잘 알고 만든건 아니고, 웹디자이너로 근무하면서도 HTML, CSS는 그냥 정말 기본만 알고 일했었는데… 특히 CSS는 변화가 많아서 더욱이 잘 모르겠는 상황이다. 그럼에도 불구하고 따로 공부하지 말자고 생각한건, JS에 집중하자라는 결론 때문이다. (+ Vue or React) 마침 수업시간의 다음 과목은 JS(씐난다!)이다. HTML과 CSS는 JS로 뭔가 만들면서 활용할 수 있는 범위로, 필요할 때마다 필요한 부분을 학습하는 방식으로 공부하기로 결정했다.

  3. 포비의 git을 이용한 코드리뷰 동영상은 진짜 너무 좋았다. git add / commit / clone / fork / pull request는 해봤었는데 branch clone / add upstream / fetch upstream / rebase 하는건 처음 알았다. 길지 않은 동영상인데 내가 원하던(지금까지 잘 몰랐던) 핵심은 다 있었다. 모두에게 추천

  4. 아아..빨리 python을 공부해야지! python으로 해볼 수 있는 강의나 문제들이 너무 많다. 지금은 자바밖에 못한다. 교육센터 커리큘럼의 마지막이 python인데 그 때로 미루지 말고 시간내서 python 빠르게 훝어보고 Django로 웹서비스 개발하기 같은 강의 봐야지!

  5. 페이스북에서 위의 글을 봤다. 저 글을 보고나니 역시 이 과정이 끝나면 다시 코드스쿼드로 돌아가고 싶다. 동영상들을 보고나니 더욱 그렇다.마스터즈 코스 모집 일정을 보니 레벨3이 5월7월, 레벨4가 7월9월이다. 과정이 끝나면 6월인데.. 너무 고민이 된다. 레벨 4에 들어갈 수 있을까.. 취업을 빨리 하는게 가장 좋은 걸까.. 둘중에 어떤 선택을 하던 지금은 지금하는 것을 열심히 하는 수밖엔 없다. 국비지원 과정을 선택한 것은 선택할 때는 그것이 최선(프로그래밍에 익숙해지는 시간을 벌고, 비용도 절감하는)이였고, 중간에 그만둬야 하나 고민도 했지만, 많이 배우고 있는 것도 사실이다.

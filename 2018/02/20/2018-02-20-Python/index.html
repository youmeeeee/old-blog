김왼손의 미운코딩 새끼

2018-02-20

(4시간 만에 끝내는 파이썬 기초)

  • 변수 : 대문자 소문자 구분한다. (python, PYTHON, Python 다 다름)

  • 입력 print() / 출력 input()

  • 자료형 : 숫자형 / 문자형 / 불린(True/False 대문자주의!) , 자료형 확인은 type()

  • list = [] #리스트 / tuple = () #수정 불가 리스트 / dictionary = {key1: val1, …} #map 같은 건가… 수정 가능

  • 자료형 변환하기 int() / float() / str()

  • 주석 #

  • 문자열

    • %s / %f / %d
    • {}.format()
    • 문자열은 “ “ (큰따옴표), ‘ ‘(작은따옴표) #큰 따옴표를 세개하면 줄바꿈 문자가 들어간다.
    • 문자열 Indexing은 0부터 시작, 하지만 반대로 제일 끝부터 -1로 시작할 수 있다.
    • slicing
      • [1:3] : 1~2번 인덱스
      • [:3] 처음~2번 인덱스
      • [2:] 2~끝 인덱스
    • string.split()
    • docstring
    • print(‘ ‘, end=’ ‘)
      • print() 메소드는 기본적으로 enter가 들어간다.
      • print(‘contents’, end=’’) 하면 아무것도 안들어간다.
      • 출력의 끝을 정할 수 있다.
    • Escape Code
      • \n 줄바꿈
      • \t 탭
    • ; (세미콜론) : 파이썬은 한줄에 한문장만 써는게 권장된다. 한줄에 여러개 문장을 쓸때는 ; 쓰면 된다. (권장하지 않음)

  • list

    • immutable(값을 변경할 수 없다.) / mutable(값을 변경할 수 있다.)
    • 문자열 immutable / 리스트 mutable
    • list.append() 값 추가하기
    • list.insert() #찾아보기
    • list.pop() #찾아보기
    • Indexing : 문자열 Indexing과 동일
    • del list[index] : 해당 인덱스 값 삭제하기
    • Slicing : list[1:3] 이면 1~2번까지 인덱스
    • list.sort() : 오름차순으로 정렬
    • list.count() : list.count(‘value’) : 리스트 안에 value가 몇 개 있는지 카운트 하는 메소드
    • len(list) : 전체 리스트 사이즈 리턴하는파이썬 내장함수
    • 함수와 메소드는 차이가 있다.

  • tuple

    • 리스트와 비슷하나 값을 변경할 수 없다. immutable.
    • 리스트는 [], 튜플은 (), 심지어 () 안써도 된다.
    • packing
      • my_tuple = 1, 2, 3
    • unpacking
      • num1, num2, num3 = my_tuple
      • num1 = 1 / num2 = 2 / num3 = 3
    • (unpacking) num 1, num2 = num2, num1 (packing)
      • num1 = 2 / num2 = 1

  • for

    • for 변수 in 컨테이너:
      실행할 명령1
      실행할 명령2
      ( : 와 4칸 들여쓰기로 코드블럭을 구분. 필수!)
    • range() 파이썬 내장함수
      • range(3) 은 range(0, 3), list=[0,1,2]과 같다.
      • for n in range(3):
        ​ print(n)
        하면 0,1,2가 출력된다.
    • for x 2
    • Comprehension : 어려운데 많이 쓰인다.

  • 연산자

    • 할당 연산자 : =, +=, -=, *=, /=
    • 산술 : +, -, , /, * 제곱, // 몫, % 나머지 연산자
    • 문자열 연산자 : +, * (숫자/ 숫자만큼 문자를 반복함)
    • 비교연산자 : ==, !=, > , < , >=, <=
    • 논리연산자 : and, or, not(뒤집기)
    • 멤버쉽 : in, not in (배열에 포함된 요소인지 아닌지 체크해주는 연산자 True, False 리턴)

  • 조건문

    • if 조건 :
      ​ 실행할 명령 1
      ​ 실행할 명령2
      ​ ( : 와 4칸 들여쓰기로 코드블럭을 구분. 필수!)
      elif 조건:
      ​ 실행할 명령1
      ​ 실행할 명령2
      else 조건 :
      ​ 실행할 명령1
      ​ 실행할 명령2

  • while

    • while 조건:
      ​ 실행할 명령1
      ​ 실행할 명령2
    • continue, break

  • 딕셔너리 (Map Collection이랑 비슷함)

    • {key1: val1, …}
    • (추가) my_dict[‘key’] = ‘value’
    • (삭제) del my_dct[‘key’]
    • 딕셔너리 메소드
      • (value가져오기) my_dict.values()
      • (key 가져오기) my_dict.keys()
      • (key, value 같이 가져오기) my_dict.items()

  • 함수

    • 내장함수
    • 모듈함수 : import해서 써야하는 함수들
    • 사용자 정의 함수
      • def 함수이름(인자, …):
        ​ 실행할 명령1
        ​ 실행할 명령2
        ​ return 결과
      • 여러개 돌려주기

  • 모듈

    • 비슷한 함수들을 모아둔 파일? 자바에서 class개념인가..?
    • 랜덤(One of module)
      • random.choice()
      • random.sample()
      • random.randint()

  • 객체 : 파이썬에서는 모든것이 객체다.

  • 코딩스타일

    • PEP8 파이썬에서 권장하는 컨벤션

      A Foolish Consistency is the Hobgoblin of Little Minds.

ReactJS로 웹서비스 만들기 DAY1

Nomade Academy

  • 윈10에서 Ubuntu Bash shell 사용할 수 있게 설정 (참고페이지)

  • npm/ Node.js / Yarn install

    • 필요한 것들을 전부 설치했는데 bash shell에서는 동작을 안했다. 참고페이지 보고 bash shell에서 다시 설치하니 되긴 된다. 흠. 근데 create react app install하다가 또 오류. 이번엔 스택오버플로우 참고. 그래도 안되서 전부 sudo에서 설치와 실행. 이 부분 잘 모르겠다. 맥사고 싶다. 리눅스 커맨드 공부는 따로..

  • React는 프레임워크가 아니라 UI Library이다.

  • 여기서 사용하는 코드는 리액트 코드이기 떄문에 이걸 자바스크립트로 바꿔주는 툴이 필요하다. → 트랜스파일러 → 웹팩(리액트 코드를 브라우저가 이해할 수 있는 코드로 바꿔준다.)

  • 자바스크립트의 가장 최신은 JS6(ES6, ECMAScript6). 모든 브라우저가 ES6을 이해하지 못하기 때문에 웹팩을 이용해서 JS를 브라우저가 이해할 수 있게 변경해줘야한다.

  • 즉, 리액트로 작업하려면 webpack과 같은 모듈번들러가 필요하다. 하지만 이 강의에선 시간이 없기떄문에?! 페이스북에서 초보자들을 위해 재공하는 create react app(CRA)을 사용한다. 웹 팩과 같은 툴을 사용할 필요없이 손쉽게 리액트 앱을 만들어주는 툴이다.

  • pre-built development server

  • 리액트는 components 컴포넌츠에 기반하고 있다. 이번 실습에선 무비리스트 / 영화카드 컴포넌트 / 이미지 컴포넌트가 있다.

  • 자바스크립트 안에 html이 자리잡고 있다. 이것을 JSX라고 한다. JSX는 리액트 컴포넌트를 만들때 사용하는 언어다.

  • 컴포넌트는 각기 다른 functions와 methods를 가지고 있다. 그걸 다 지금 배울건 아니고 하나만 기억하기! 모든 컴포넌트는 render function을 가지고 있다. renden의 기능은 뭔가를 보여주는, 출력하는 기능이다. 이 컴포넌트가 나에게 보여주는 것이 무엇인가! 이게 render의 기능이다.

  • 우리는 index.js에 파일을 만들고 있다. index.js는 react, reactDOM, css, (가장 큰)컴포넌트’app’ 을 import한다.

  • react와 reactDOM의 차이점. react는 UI 라이브러리다. reactDOM은 리액트틀 웹사이트에 출력(render)하는 걸 도와주는 모델이다. DOM(Document Object Model). 리액트를 사용해서 웹사이트에 올려놓고 싶다 할때 reactDOM을 쓰면 된다. 리액트를 모바일 앱에 올려놓고 싶을때는 reactNative를 쓴다. 리액트돔은 1개의 컴포넌트를 출력(render)하고, 그 다큐먼트 안에 엘리먼트가 있는데, 엘리먼트 아이디는 root이다. 이건 index.html에 있다.

  • 컴포넌트 생성 > 렌더 > 리턴 > html내용 > 브라우저에서 확인

  • import react > class > RENDER( 필수기능)

  • 큰 컴포넌트 안에 작은 컴포넌트를 작은 컴포넌트를 집어 넣는 방식으로 작업한다.

  • 모든 컴포넌트는 render 해야하고 return해야 한다.

  • 컴포넌트 > render > return > JSX (is the way rewrite HTML on react)

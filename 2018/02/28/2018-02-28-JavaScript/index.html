DOM(1)

  • (X)HTML 문서 구조 (Document Object Model)
  • (X)HTML 각 태그마다 기능과 속성이 있다. 예를 들어 img태그는 문서에 이미지를 출력하는 기능(method)이 있고 속성(properties)으로는 src와 alt를 포함하고 있다.
  • DOM을 배우는 주된 목적은 자바스크립트를 사용하여 문서 객체를 선택하고 속성 또는 CSS를 적용하기 위해서이다.
  • JS DOM은 IE8 이하버전과 파이어폭스, 크롬, 사파리 등의 브라우저 간에 호환성이 떨어진다는 단점이 있다.
  • DOM을 그림으로 표현하면 트리구조로 되어있다. html은 뿌리가 된다. 이 뿌리로 부터 가지처럼 뻗어가는 것을 노드(node)라고 부른다. node에는 X(HTML) 태그를 연결하는 Element node, 텍스트를 연결하는 Text node, 그리고 (X)HTML 태그에 속성을 연결해주는 Attribute node가 있다.
  • 선택자
    • 직접문서에서 요소를 선택해 오는 직접 선택자(id, class, form name, Element Name 포함)이 있다.
    • 이 선택자를 사용해 선택해 온 문서 객체를 기준으로 가까이에 있는 요소를 선택하는 인접 관계 선택자(parentNode, childeNode, firstChild, children, previousSibling, nextSibling 선택자를 포함)가 있다.
    • 쉽게 말하면, 먼거리에 있는 요소를 선택해오는 원거리 선택자(직접 선택자)가 있고, 선택한 요소를 기준으로 가까운 거리에 있는 요소를 선택하는 근거리 선택자(인접 관계 선택자)가 있다.
    • 인접 관계 선택자를 사용하기 위해서는 반드시 인접 요소의 관계를 잘 파악하고 있어야 한다. 그래야 이미 선택한 문서 객체를 기준으로 인접한 문서 객체를 정확히 선택할 수 있다.
    • 선택자 종류
      1. 직접 선택자
        • document.getElementById(“아이디명”)
        • document.getElementByTagName(“태그명”)
        • document.formName.inputName
      2. 인접 관계 선택자
        • parentNode : 선택한 요소의 부모요소를 선택해온다.
        • childeNode : 선택한 요소의 모든 자식 요소를 선택해 온다. 선택한 모든 요소가 배열 객체로 저장된다.
        • children : 선택한 요소의 자식 요소인 태그만 선택해온다. 선택한 모든 요소가 배열 객체로 저장된다.
        • firstChild : 선택한 요소의 첫번째 자식 요소만 선택해 온다.
        • previousSibling : 선택한 요소의 이전에 오는 형제 요소만 선택해 온다.
        • nextSibling : 선택한 요소의 다음에 오는 형제 요소만 선택해 온다.
        • perentNode, firstChild, previousSibling, nextSibling은 공백문자 인식 호환성에 문제가 있다.

선택자 적용 위치와 방법

직접 선택자와 인접 관계 선택자를 사용할 때는 head 태그 영역에 자바스크립트를 선언하게 되면 (X)HTML을 loading 해오는 것보다 스크립트의 선택자가 먼저 실행되어 문서 객체를 선택할 수 없게 된다.

X(HTML) 문단 태그보다 먼저 자바스크립트가 문서 상단에 위치해 있어 먼저 실행되면 다음에 오는 문단 태그를 스크립트의 선택자를 사용해서 선택해 올 수 없게 된다.

선택자 사용시 잘못된 적용 위치

스크립트에 선택문을 사용할 경우에는 body 태그 영역에 문단 태그 작성을 마친 후, 마지막 부분에 작성하거나, 아니면 window.onload = function(){…}을 이용하여 선택문을 작성해야 한다. 여기서 window.onload=function(){…}은 모든 문서의 객체를 로딩한 후에 중괄호 {…} 내에 일련의 실행문을 실행하시오라는 명령어이다.

노드 타입으로 인접 관계 선택자 문제 해결하기

DOM에서 인접 관계 선택자를 사용하면 IE 8 이하 브라우저와 그 이외의 브라우저에서 호환성 문제가 생긴다는 것을 확인했다. 이 호환성 문제는 노드 타입(nodeType)을 이용하면 해결할 수 있다.

노트 타입

(X)HTML의 노드에는 (X)HTML 태그를 연결하는 Element(요소)노드와 텍스트를 연결하는 Text노드, (X)HTML 태그에 속성을 연결해주는 Attribute(속성)노드가 있다.

<노드 타입의 종류>

종류 타입값
ElementNode (요소노드) 1
AttributeNode (속성노드) 2
TextNode (텍스트노드) 3

예제 1

다음 문단 태그에 인접 관계 선택자를 사용하여 요소를 선택하고, 선택한 요소는 노드 타입을 IE 8 이하 버전의 브라우저와 그 밖의 브라우저에서 확인해보자.

IE 8 이하 브라우저

↑ IE 8 이하 버전에서 실행하면 줄바꿈을 해도 공백 문자가 생성되지 않으므로 정상적으로 내용3의 p태그가 선택된다. 그 결과로 노트 타입값을 검사하면 1을 리턴한다.

파이어폭스, 크롬, 사파리, 오페라 등의 브라우저

↑ 위 브라우저들은 다음과 같이 인접 관계 선택자를 실행하면 X(HTML)의 코드에 줄바꿈이 일어날때마다 공백문자가 생성되어 p태그가 아닌 공백문자가 선택된다. 그 결과 노트 타입 값을 검색하면 1이 아닌 3이 반환된다.

공백문자가 아닌 요소 노드를 정상적으로 선택하려면, firstChild 첫번째 자식 요소 선택자의 경우, 다음 ‘비표준 인접 관계 선택자 해결’ 의 예제와 같이 childen[0] 할 수 있다. children[0]은 텍스트 노드를 무시한 채 선택하기 때문에, 공백문자는 선택되지 않고 오로지 요소 노드만 선택된다. 그리고 nextSibling의 경우에는, 노드타입의 값을 검사해서 3이 나옴녀 계속해서 1이 나올때가지 다음 nextSibling을 하게 만든다. 다음 ‘비표준 인접 관계 선택자 해결’ 예제와 같이 반복문 while을 잉요하면, 노드 타입의 값이 1이 아닐 경우 계속 해서 다음요소가 선택된다. 즉 변수 p에 노드 요소를 참조해야만 조건식에 노드 타입의 값이 1이 되어 반복문을 종료하게 된다.

this

일반적으로 메소드를 호출한 객체(다…)가 저장되어 있는 속성이다. 메소드를 호출할 때 만들어진다.

this의 정체…

this를 잘 알려면 실행 컨텍스트에 대해서 알아야한다. (*실행 컨텍스트 : 자바스크립트 엔진이 구문을 해석하는 단위로 크게는 함수 실행 컨텍스트, 전역 실행 컨텍스트가 있다.)(찾아보기?실행컨텍스트?변수객체화?호이스팅?)

  • 일반 함수
    일반함수에서는 this는 window가 된다. 함수가 호출이 될 때 만들어진다.

  • 중첩 함수
    중첩함수에서의 this도 일반 함수와 동일하기 window가 된다.

  • 이벤트 리스너

  • 메소드

  • 메소드 내부의 중첩 함수

함수호출() vs new함수호출()

함수단위 코딩 vs 클래스 단위 코딩

함수단위 코딩 단점

  • 함수 방식 클래스와 동일하게 함수가 중복해서 만들어 진다.
  • 함수 외부에서 함수 내부 속성과 함수에 접근 할 수가 없다.

클래스 단위 코딩은 위의 단점의 개선이 가능하다.

인스턴스 프로퍼티와 클래스 프로퍼티

인스턴스 프로퍼티 - 인스턴스를 생성 해야지만 사용 할 수 있는 프로퍼티. 객채의 상태정보를 담거나 다뤄야 하는 경우에 주로 사용한다.

클래스 프로퍼티 - 인스턴스를 생성 하지 않고도 사용할 수 있는 프로퍼티. (자바에서 static 같은 건가… es5 이후로는 static이 나온다고 한다.) 독립적으로 실행 가능한 유틸리티성 정보와 기능을 구현할 때 주로 사용한다.

패키지

JS는 패키지 문법을 제공하지 않지만 흉내는 낼수 있다. 오브젝트 리터럴을 이요한다. 프로퍼티와 메솓가 전혀 없는 빈 오프젝트를 만들어 클래스들을 담아 사용한다.

var packageName = {}

var packageName = new Object();

packageName.className = function() {}
